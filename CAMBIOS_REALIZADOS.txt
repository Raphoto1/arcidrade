┌─ PROBLEMA IDENTIFICADO ────────────────────────────────────────────────────┐
│ Errores intermitentes "failed to connect to upstream database" (P1001)     │
│ que aparecen y desaparecen aleatoriamente en Vercel.                      │
└────────────────────────────────────────────────────────────────────────────┘

┌─ CAUSA RAÍZ ──────────────────────────────────────────────────────────────┐
│ • Timeouts de conexión insuficientes (default: 5 segundos)                 │
│ • Saturación del connection pool bajo carga                                │
│ • Problemas transitorios de red (Vercel → DB)                             │
│ • Falta de retry logic para errores transitorios                           │
└────────────────────────────────────────────────────────────────────────────┘

╔════════════════════════════════════════════════════════════════════════════╗
║                         1. CAMBIOS EN CÓDIGO                              ║
╚════════════════════════════════════════════════════════════════════════════╝

✅ src/utils/db.ts
   ├─ Pool configuration optimizada:
   │  ├─ max: 20 → 15 conexiones (menos contención)
   │  ├─ min: 2 → 1 (menos overhead)
   │  ├─ connectionTimeoutMillis: 10s → 20s (permite reconexiones lentas)
   │  ├─ idleTimeoutMillis: 60s → 30s (detecta stale connections)
   │  ├─ statement_timeout: 60s → 30s (falla rápido en queries lentas)
   │  └─ Nuevo: idleErrorTimeout: 5000 (limpia conexiones con error)
   │
   └─ Error handling mejorado en setupPoolErrorHandling()

✅ src/utils/retryUtils.ts
   ├─ Nueva función: isTransientConnectionError()
   │  └─ Detecta P1001 y otros errores transitorios
   │     • P1001: "failed to connect to upstream database"
   │     • ECONNREFUSED, ETIMEDOUT, socket hang up
   │     • Connection timeout, EHOSTUNREACH, ENETUNREACH
   │
   ├─ Mejorado: withRetry() con P1001 detection
   │  ├─ Falla inmediatamente si error es permanente
   │  ├─ Reintenta solo si error es transiente
   │  └─ Logging mejorado: indica si error es transitorio
   │
   └─ Estable: withPrismaRetry()
      └─ 3 intentos, backoff: 100ms → 200ms → 400ms

   4. Vercel automáticamente redeploy (1-2 minutos)
